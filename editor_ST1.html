Volunteer Page

The Volunteer Page promotes volunteering programs with interactive cards, a rating summary, and a feedback form.

Technical Implementation

HTML Structure: Used <section> for scrollable program cards, ratings, and feedback form. Each card uses <article> with <img> (royalty-free from iStockPhoto), headings, and buttons. Included a non-functional <select> for sorting and a feedback form with <textarea>, <button>, and emoji ratings.

CSS Styling: Applied global.css for layout (--space-4 padding) and colors (--primary-500 for buttons). Used CSS Grid for card layout and hover effects (scale transform, --shadow-md). Styled buttons with .btn-primary and .btn-secondary classes, adding transition for interactivity.

Purpose: Semantic structure enhances accessibility, while hover effects and grid layout improve user engagement. The design uses health-themed colors to maintain consistency.

Validation: Validated via W3C HTML Validator. See validation results.

Challenges & Solutions

  • Challenge: Splash Screen auto-redirect timing was too fast. Solution: Adjusted <meta> refresh to 4 seconds for better user experience.
  • Challenge: Volunteer page cards overflowed on mobile. Solution: Used CSS Grid with minmax and media queries for responsive layout.
  • Challenge: Content page images distorted on smaller screens. Solution: Applied object-fit: cover and responsive max-width.

Lessons Learned: Testing across devices early and using semantic HTML improves accessibility and maintainability.

Accessibility

All pages meet WCAG 2.1 AA standards:

  • Semantic HTML: Used <header>, <main>, <section>, and <article> for structure.
  • Keyboard Navigation: Added tabindex="0" to interactive elements and skip-to-content link.
  • Color Contrast: Ensured 4.5:1 ratio using --primary-600 and --neutral-50.
  • Alt Text: Provided descriptive alt text for all images and icons.
  • Reduced Motion: Supported via @media (prefers-reduced-motion) in global.css.
